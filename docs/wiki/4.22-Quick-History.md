# Quick History

Quick History has been **merged into the unified [[4.21-Worklog|History]] view**. It is no longer a separate, current-year, week-centric screen, and there is no mode toggle: the **quick-history route** (e.g. `project/:id/quick-history` or `tag/:id/quick-history`) is now a **legacy alias** that opens the same full History view as the `worklog` and `history` routes.

The unified History view shows your tasks' **time spent per day** — from both active and archived tasks, scoped to the current **work context** (project or tag) — as a hierarchy of **years → months → weeks → days → task entries**, across **all years** (not only the current one). Leaf-task time can be **edited inline** for a given day. For the full description of how this data is built, refreshed, and exported, see [[4.21-Worklog]].

## History

Quick History was originally a compact, current-year, week-first companion to the Worklog with inline time editing. The two have since been combined into a single History view, so the separate "Quick History mode" and the current-year-only limit no longer exist. The legacy route is kept so older links and bookmarks keep working.
